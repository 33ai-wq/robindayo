Welcome. This page was built on a single principle learned from real-life guests: a visitor is owed two things — respect, and value. On RobinDayo, both have a precise shape: a small on-chain settlement, transparent in price, returned instantly in the response body.

What you're looking at is a catalog of working endpoints, each hosted and billed on the x402 protocol (USDC on Base, settled automatically). There's no front-end cloud bill pushed onto you, no upsell modal, no SDK to install — just click, pay what the card shows, get your answer. Usually under a second.

How a visit works

Four steps from arrival to answer

01 · LOOK
Pick an endpoint

Read the card. Note the price and the network. Move on if it's not for you — no click, no charge.

02 · KNOCK
Click "Try it"

Your client sends a normal HTTP request. The worker answers with HTTP 402 and the exact settlement terms.

03 · PAY
Settle one penny

Your x402 client signs a USDC transfer to the address in the 402 envelope, retries with the receipt header.

04 · ANSWER
Receive the data

The worker returns the response. Receipt lives on-chain. You're a guest who paid the bill — clean, done.

Philosophy

Why this is just plain HTML

// 01

No front-end server

No Cloudflare Pages. No React. No bundler. GitHub Pages is enough for a catalog. Operating cost rounds to zero.

// 02

Guest = transaction

Every request settles as a micro-payment over x402. Stop visiting, stop paying. No auto-billing, no trapped trial.

// 03

Honourable pricing

USDC prices match x402scan. There's no RobinDayo markup. If you ever find one, that's a bug — and it gets refunded.

// 04

Source-open

The worker logic is public. This repo is the catalog; the worker sources live in adjacent repos. Audit them.

// 05

Two rails, one price

Both x402 (USDC) and L402 (Lightning sats) point at the same response. Pay the way that suits your stack.

// 06

No tracking

No analytics SDKs, no fingerprinting, no cookies. The HTTP request is the only thing the workers see.
